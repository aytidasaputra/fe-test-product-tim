Pengembangan Sistem Manajemen Asuransi
DESKRIPSI PROYEK

Buatlah sebuah aplikasi web "Sistem Manajemen Asuransi" menggunakan HTML, CSS, dan TypeScript. Anda diperbolehkan menggunakan framework JavaScript modern (React, Vue, Angular, dll) sesuai preferensi Anda.

WAKTU PENGERJAAN

Maksimal 3 hari sejak menerima tes ini.

PERSYARATAN FUNGSIONAL

1. Manajemen Polis Asuransi

  • Menampilkan daftar polis asuransi
  • Menambahkan polis asuransi baru
  • Melihat detail polis
  • Mengubah status polis (aktif/non-aktif/klaim)
  • Mengedit informasi polis

2. Fitur Polis Asuransi

  • Setiap polis harus memiliki: nama pemegang polis, jenis asuransi (kesehatan/jiwa/kendaraan), nomor polis, nilai pertanggungan, tanggal mulai, tanggal berakhir, dan status (aktif/non-aktif/klaim)
  • Kemampuan untuk memfilter polis berdasarkan jenis asuransi dan status
  • Kemampuan untuk mencari polis berdasarkan nama pemegang atau nomor polis

3. Autentikasi Sederhana

  • Halaman login sederhana (tanpa backend sebenarnya, cukup simulasikan dengan local storage)
  • Hanya pengguna yang "login" yang bisa mengakses aplikasi

4. Kalkulator Premi

Implementasikan kalkulator sederhana untuk menghitung premi berdasarkan:

  • Jenis asuransi
  • Nilai pertanggungan
  • Usia pemegang polis (untuk asuransi kesehatan/jiwa)
  • Jenis kendaraan (untuk asuransi kendaraan)

5. Persistensi Data

  • Menggunakan localStorage untuk menyimpan data polis agar tetap ada setelah halaman di-refresh
PERSYARATAN TEKNIS

Teknologi yang Wajib Digunakan

  • TypeScript untuk semua kode JavaScript
  • Unit Testing menggunakan Jest, React Testing Library, atau framework testing lainnya
  • Deployment wajib dilakukan ke Vercel

UI/UX

  • Desain yang responsif (mobile-first approach)
  • UI yang intuitif dan mudah digunakan
  • Implementasi tema gelap/terang (dark/light mode)
  • Transisi dan animasi sederhana untuk meningkatkan pengalaman pengguna

Kode

  • Organisasi kode yang rapi dan terstruktur
  • Penamaan variabel dan fungsi yang jelas dan konsisten
  • Penggunaan komentar yang tepat
  • Penanganan error yang baik
  • Type safety dengan TypeScript yang baik (minimal penggunaan any)

Bonus Point (Opsional)

  • Dashboard visual dengan grafik ringkasan polis (distribusi berdasarkan jenis, status, dll)
  • Simulasi notifikasi untuk polis yang akan berakhir dalam 30 hari
  • Test coverage di atas 70%
  • Fitur pengajuan klaim sederhana
  • Ekspor data polis ke format PDF atau Excel (simulasi)
